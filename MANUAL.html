Section 1

What each file does

Luminary is just three files. That's the entire application. Here's exactly what each one is and who uses it:

📋
send.html
The Sender's Dashboard
This is the page the person giving the birthday wish opens. It contains a 4-step form where they type the birthday person's name, add recipients, write a letter, record a voice message, upload photos, and choose a theme. When they click Send, it sends a real email with a link. It also has a Dashboard tab showing all past wishes and who opened them.
Used by: Sender only
🎂
experience.html
The Birthday Experience
This is the page the birthday person opens when they click the link in their email. It shows the cinematic curtain with their name, the animated birthday cake with hand-tracking, their personalised letter, voice message, memory slideshow, age countdown, and a response panel. They never see send.html. This page reads all the personalised data from the URL link itself.
Used by: Receiver only
⚡
netlify/functions/
send-email.js
The Email Server
This is a small server-side script that runs on Netlify's servers (not in the browser). When the sender clicks "Send the Experience", send.html calls this function, which uses the Resend email API to actually deliver the email to the recipient's inbox. Without this, emails don't send automatically — the app falls back to opening Gmail or Mail app manually.
Runs on: Netlify server
⚙️
netlify.toml
Deployment Configuration
A configuration file that tells Netlify where the HTML files live (the public/ folder) and where the server functions live (netlify/functions/). Also sets up the URL routing so /api/send-email calls the right function. You never edit this file — it just needs to be present.
Infrastructure only

Full folder structure

# This is what you deploy to Netlify: luminary-prod/ ├── netlify.toml ← tells Netlify how to serve everything ├── DEPLOY.html ← deployment guide (open in browser) ├── MANUAL.html ← this file (user manual) │ ├── public/ ← all files the browser can see │ ├── send.html ← sender opens this │ └── experience.html ← receiver opens this │ └── netlify/ └── functions/ └── send-email.js ← server function (real email)

The two HTML files are completely self-contained. Every line of CSS, JavaScript, and logic is inside each HTML file — no external stylesheets, no script imports, no build step required. Open them directly in a browser and they work.

Section 2

How the whole thing works

Here is the complete journey from sender to receiver, explained step by step:

✍️
Sender opens send.html and fills the form
The sender types the birthday person's name, age, birthday date, and adds recipient email addresses. They write a letter, optionally record a voice message, upload photos, and choose a visual theme. All of this happens in the browser — nothing is sent to any server yet.
🔗
All data is packed into a single URL
When the sender clicks Send, JavaScript takes everything they entered — name, letter, photos, voice recording, theme, candle count — compresses the photos, and encodes it all into one long URL parameter. This is the experience link.
https://yoursite.netlify.app/experience.html?d=eyJuYW1lIjoiWmFyYSIs...
📧
A real email is sent to the recipient
send.html calls the Netlify serverless function (send-email.js), which uses the Resend email API to send a beautiful branded HTML email to each recipient. The email contains the experience link as a button. This happens automatically — no manual clicking, no mail app needed.
POST /api/send-email → Resend API → recipient's inbox
📩
Recipient receives the email
The birthday person gets a beautiful email with their name in the subject line, a warm message, and a gold button saying "Open Your Birthday Experience". The email works on any email client — Gmail, Outlook, Apple Mail, Yahoo.
🎭
Receiver clicks the link — experience loads
experience.html opens in their browser. It reads the ?d=... parameter from the URL, decodes it, and rebuilds the entire personalised experience — their name, the letter, the photos, the voice message, the candle count, the theme. Everything is already in the link. No server request needed.
✨
The birthday party plays out
A cinematic curtain shows their name. They click in, the camera activates, they light candles with their finger, all candles lit reveals their personalised wish, the slideshow plays with Ken Burns effects, the letter appears in elegant italic type, the voice message plays. They can capture polaroid moments and send a thank-you response.
📊
Sender sees it was opened
When the recipient opens the link, experience.html writes an "opened" timestamp to the browser's local storage. The sender can check the Dashboard tab in send.html to see who received and who opened each experience.

The key insight: The experience link is self-contained. It doesn't phone home to any server when the recipient opens it. The entire birthday party lives inside the URL. This is why it works from any device, any browser, any country.

Section 3

Sender — full walkthrough

You are the sender if you want to create and send a birthday experience to someone. Here is every step in detail.

Opening the app

Open send.html in your browser. If running locally, double-click the file or open it via a local server. If deployed, go to https://your-site.netlify.app/send.html.

You'll see the Luminary dashboard with two tabs at the top: ✦ Compose and ◈ Dashboard. Start on Compose.

Step 1 of 4 — About the birthday person

1
Enter their name (required)
This appears on the curtain intro, in the wish message, in the email subject, and throughout the experience. Use their first name or nickname.
2
Enter their age (optional)
Used on the experience page to show how many days they've been alive. Leave blank if you don't want to reveal their age.
3
Enter their birthday date (optional)
Powers a live countdown to their next birthday, ticking to the second. Leave blank to hide the countdown.
4
Add recipient emails (required — at least one)
Type an email address in the field and either press Enter or click + Add. Repeat for each recipient. You can add as many as you like.

Each email appears as a chip below the field with an × to remove it.

Important: You must click + Add (or press Enter) for each email. Simply typing an address and clicking Continue will auto-add it for you, but it's clearer to click + Add explicitly.
5
Choose wish language
Select English, Bengali, Spanish, Arabic, or French. The personalised birthday wish that appears after all candles are lit will be in this language.
6
Click Continue →
The progress bar at the top advances. If name or email is missing, a red error message appears below the relevant field.

Step 2 of 4 — Your message

1
Your name (optional)
Shown on the experience as "Arranged with love by [your name]" and signed at the bottom of the letter.
2
Write a heartfelt letter (optional)
This appears in elegant Cormorant Garamond italic font on the experience page with a gold left-border accent. Write as much or as little as you like. The field scrolls if the letter is long.
3
Record a voice message (optional)
Click the ⏺ circle button to start recording. Your browser will ask for microphone permission — click Allow.

Speak your message. Click ⏹ to stop. Maximum 2 minutes.

Click ▶ Play to hear it back before sending. Click ✕ to delete and re-record.

The voice recording is encoded into the URL so it travels with the link.
4
Set number of candles
Tap − or + to set 1–14 candles on the birthday cake. Matching the person's age is a nice touch.

Step 3 of 4 — Memories & atmosphere

1
Upload memory photos (optional)
Click the upload zone to open a file picker. Select multiple photos at once using Ctrl+Click or Cmd+Click.

Photos are automatically compressed to max 480px and JPEG quality 72% to keep the URL manageable. Thumbnail previews appear below.

Recommended: 3–6 photos. More than 8 may make the URL too long for some email clients. A warning appears if photos exceed 800KB combined.
2
Choose an atmosphere theme
Six options: Midnight (dark gold), Golden Hour (warm amber), Aurora (teal/purple), Cherry Blossom (rose/pink), Deep Ocean (blue), Cosmos (purple/orange). This changes the ambient glow colour throughout the experience.

Step 4 of 4 — Review & send

1
Review the envelope preview
Shows the birthday person's name, a preview of the letter, candle count, theme, photo count, voice status, and all recipient emails. Check everything looks right.
2
Optionally customise the email subject
Default: "A birthday experience is waiting for you, [Name] 🎂". Change it to anything you like.
3
Click ✦ Send the Experience
If deployed on Netlify with Resend configured: email sends automatically to all recipients. You'll see "✅ Sent to X recipients!" on the success screen.

If running locally: shows Gmail and Mail App buttons for manual sending.

Section 4

Receiver — full walkthrough

You are the receiver if you got an email with a birthday experience link. Here is everything you'll see and do.

Opening the email

You'll receive an email with the subject something like "A birthday experience is waiting for you, [Your Name] 🎂". Open it on any device — phone, laptop, tablet.

Inside the email is a gold button: "✦ Open Your Birthday Experience". Click it. Best on a laptop or desktop for hand tracking, but the full experience works on mobile too (just click candles instead).

The curtain intro

The page opens with a full-screen black curtain showing your name in large letters with a gentle gold glow. Floating hearts drift upward. A gold line expands across the screen.

Click the "Open Your Experience ✦" button when you're ready. This is intentional — it gives you a moment to prepare before the party begins.

The birthday experience

1
Allow camera access
Your browser will ask for camera permission. Click Allow. The camera is used only for hand tracking — it runs entirely in your browser. Nothing is recorded or uploaded anywhere.
2
Light the candles with your finger
Look at the camera panel on the left. You'll see your live camera feed with a gold skeleton overlay on your hand.

Point your index finger (the one you use to point at things) at a candle on the cake. Move it slowly. When your fingertip gets close to a candle wick, it lights automatically with a sound.

Tip: move slowly and keep your hand 30–60cm from the camera. Good lighting helps.
3
Blow out candles (optional)
Hold your open palm flat, facing the camera with all 4 fingers fully extended. The "Blowing 〰" badge appears and all candles go out with a whoosh sound. Then you can light them again.
4
Light ALL candles to reveal your wish
When the counter at the bottom of the cake shows all candles lit (e.g. "5 / 5"), your personalised birthday wish appears in a glowing card with confetti and music. This is the moment.
5
Explore the rest of the experience
Scroll down to find:
  • Age panel — how many days you've lived, countdown to next birthday
  • Letter — the heartfelt message from the sender in italic serif
  • Voice message — click ▶ to hear the sender's voice
  • Memory slideshow — photos with cinematic Ken Burns zoom effects and sparkles
  • Capture moments — click ⊙ to take selfies that appear as polaroids
6
Send a thank-you response (optional)
At the bottom of the page, a "Send a Thank You 💛" panel appears. Choose an emoji reaction and optionally write a short note. Click Send. This is optional — the experience works perfectly if you skip it.
7
Save a screenshot keepsake
Click "↓ Save Keepsake" at the top right. This downloads a high-resolution PNG of your entire birthday experience page as a memory.

Section 6

Hand tracking guide

How it works

Luminary uses MediaPipe Hands — Google's on-device AI model that detects 21 points on your hand in real-time. Everything runs in your browser. Your camera feed is never sent to any server.

Lighting candles

✓
Point your index finger at a candle
Keep your other fingers curled. Move slowly. When the gold fingertip dot (shown on the cake) comes within 27px of a candle wick, it lights automatically.

Blowing out candles

✓
Open palm, all 4 fingers extended
Hold your hand flat like you're pressing it against a wall. All four fingers (index, middle, ring, pinky) must be above their knuckles. The "Blowing 〰" badge appears and all candles extinguish.

Tips for best results

ProblemSolution
Hand not detectedImprove lighting. Ensure your hand contrasts with the background.
Candle won't lightMove your finger more slowly. Keep your hand 30–60cm from camera.
Wrong finger lighting candlesCurl all other fingers. Only the index fingertip (landmark 8) triggers lighting.
Blow not workingExtend all 4 fingers completely. Don't just wave — hold them extended.
Camera shows blackCheck browser permissions. Click the camera icon in the address bar → Allow.
Camera works on phone but not laptopSome laptops block camera under http://. Needs https:// — deploy to Netlify.

Section 7

Running locally

Running locally means opening the files on your own computer without deploying anywhere. This is fine for building and previewing — but the experience link will show 127.0.0.1 and won't work for recipients.

What works locally
✅Form fills and validates correctly
✅Preview experience (inline iframe)
✅Voice recording and playback
✅Photo upload and compression
✅Dashboard and tracking UI
✅Camera on localhost (most browsers)
❌Link works for recipient
❌Automatic email sending
What works after deployment
✅Everything above
✅Link works for anyone worldwide
✅Emails send automatically
✅Camera on all browsers (HTTPS)
✅Beautiful branded email template
✅Multiple recipients at once

Method A — Double-click (simplest)

1
Put both HTML files in the same folder
send.html and experience.html must be in the same folder. The preview function uses a relative path to load experience.html.
2
Double-click send.html
Opens in your default browser at a file:// address. Fill the form and use the inline preview to see the experience. Note: camera may be blocked on some browsers under file://.

Method B — Local server (recommended for full features)

A local server gives you http://localhost instead of file://. Camera works reliably. Preview works perfectly.

Option 1: VS Code Live Server (easiest)

1
Install VS Code + Live Server extension
Download VS Code. Open Extensions (Ctrl+Shift+X). Search "Live Server" by Ritwick Dey. Install.
2
Open the folder in VS Code
File → Open Folder → select your luminary folder.
3
Right-click send.html → Open with Live Server
Browser opens at http://127.0.0.1:5500/send.html. Everything works including camera.

Option 2: Python (built into most computers)

# Open terminal, navigate to your luminary folder cd path/to/luminary-prod/public # Start local server python3 -m http.server 8080 # Open in browser: http://localhost:8080/send.html

Option 3: Node.js serve

# Install serve (one-time) npm install -g serve # Run from your public/ folder serve . # Opens at: http://localhost:3000/send.html

Reminder: However you run it locally, the experience link will always show your local address (127.0.0.1 or localhost). That link only works on your computer. To share with a real recipient, deploy to Netlify (Section 8).

Deploying online

Deployment means putting the files on a public server so anyone can access them. Takes 5 minutes. Free.

Deploy to Netlify (recommended)

1
Create a free Netlify account
Go to netlify.com → Sign up. Free forever, no credit card.
2
Go to the deploy page
Go to app.netlify.com/drop or click "Add new site" → "Deploy manually" in the Netlify dashboard.
3
Drag the luminary-prod folder onto the drop zone
Drag the entire luminary-prod folder (which contains netlify.toml, public/, and netlify/) onto the Netlify drop zone. Wait ~20 seconds.
4
You're live!
Netlify gives you a URL like https://luminous-abc123.netlify.app. Open https://your-site.netlify.app/send.html — it works. Share the experience links and they work for anyone.

Section 9

Email setup (Resend)

Without this setup, clicking Send shows Gmail/Mail app buttons. With this setup, emails go out automatically the moment you click Send.

1
Sign up at resend.com
resend.com → Sign up free. 3,000 emails/month free. No credit card.
2
Get your API key
Resend dashboard → API Keys → Create API Key → name it "Luminary" → Copy the key (looks like re_aBcDeFgH_...). Save it — shown only once.
3
Add environment variables in Netlify
Netlify dashboard → your site → Site configuration → Environment variables → Add a variable:

RESEND_API_KEY = your key from step 2
FROM_EMAIL = Luminary <onboarding@resend.dev> (no domain setup) or Luminary <birthday@yourdomain.com> (with domain)
4
Redeploy Netlify
Netlify dashboard → Deploys → Trigger deploy → Deploy site. Environment variables only activate after a fresh deploy.
5
Test with your own email
Create an experience with your own email as recipient. Click Send. Check your inbox. Should arrive within seconds.

Section 10

FAQ & troubleshooting

The Continue button doesn't work

Make sure: (1) the birthday person's name is filled in, (2) you've added at least one recipient email by clicking + Add. If you typed an email but didn't click Add, clicking Continue will auto-add it. Red error messages appear below each problem field.

The link shows 127.0.0.1 — recipient can't open it

You're running the app locally. The link only works on your own computer. Deploy to Netlify (Section 8) to get a real shareable link. A yellow warning banner appears on the success screen when this is the case.

Email not sending automatically

This requires Netlify deployment + Resend API key configured (Section 9). On localhost, the app falls back to Gmail/Mail App buttons. After deployment with Resend, it's fully automatic.

Camera not working

Check: (1) Did you click Allow when the browser asked for camera permission? Look for a camera icon in the address bar. (2) Are you on https://? Camera is blocked on file:// and sometimes http://. Deploying to Netlify solves this. (3) Try Chrome or Edge — they have the best MediaPipe support.

Candles won't light from my finger

Move your index finger more slowly. The detection runs at 30fps so very fast movements are missed. Keep 30–60cm distance. Good lighting matters — try near a window or a lamp.

Photos not appearing in the experience

Too many/large photos can make the URL too long. Stick to 3–6 photos. A warning banner appears in Step 3 if photos exceed 800KB combined. Delete some thumbnails and try again.

Voice message not playing

Voice is stored as audio/webm. Record using Chrome for broadest compatibility. On Safari, try recording a short test message first.

How do I send to multiple people?

In Step 1, add each email separately using the + Add button. All recipients receive the same experience link with the same personalisation. After sending, Gmail/Netlify sends to each one individually.

Can the birthday person share the link?

Yes. The link is completely portable. Anyone with the link can open the experience. You can also paste it into WhatsApp, Telegram, iMessage, or any messaging app.

Is camera data stored anywhere?

No. The camera feed is processed entirely in the browser by MediaPipe. Nothing is uploaded, stored, or transmitted. The camera is used only to detect hand landmarks in real-time.

The experience link is very long — is that normal?

Yes. All the personalisation data (letter, photos, voice, settings) is encoded directly into the URL. This is what makes it work without any server. The link can be up to 2MB long but works in all major browsers. WhatsApp and most email clients handle it fine.
