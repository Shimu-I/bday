Why nothing works on localhost

⚠ The honest truth about what's been happening

  • The link shows 127.0.0.1:5500 — that address only exists on your computer. The recipient's browser literally cannot reach it. It's like giving someone your home WiFi password and expecting them to connect from another country.
  • Emails cannot be sent from a browser — it's a security rule. No JavaScript in a browser can send an email directly. Every "send email" button you've seen just opens YOUR mail app. You still have to click Send.
  • Camera requires HTTPS — on http:// or file://, most browsers block camera access entirely.

The fix is simple: Deploy the files to Netlify (free, 5 minutes) and add a Resend API key (free, 5 minutes). After that, emails send automatically and links work for anyone in the world.

Project file structure

After downloading, your folder should look exactly like this:

# luminary-prod/ ← this is what you deploy ├── netlify.toml # tells Netlify where files live ├── public/ │ ├── send.html # sender's form page │ └── experience.html # birthday person's page └── netlify/ └── functions/ └── send-email.js # serverless function (real email)

Important: All three folders must be present. If netlify/functions/send-email.js is missing, automatic email sending won't work (it will fall back to Gmail/Mail app).

Deploy to Netlify

Netlify is free, takes 5 minutes, and gives you a real https:// URL that works for everyone.

1
Create a free Netlify account
Go to netlify.com → click Sign up → choose Sign up with GitHub or email. It's completely free — no credit card.
2
Drag and drop the folder
On the Netlify dashboard, look for "Deploy manually" or go to app.netlify.com/drop.

Drag the entire luminary-prod folder onto the drop zone. Wait about 20 seconds.

Netlify will give you a URL like: https://luminous-birthday-abc123.netlify.app
3
Verify it's working
Open https://your-site.netlify.app/send.html in your browser.
The URL in the experience link should now show https://your-site.netlify.app/experience.html?d=... — not 127.0.0.1.

That link now works for anyone in the world.

That's the most important step done. Links now work. Camera works (HTTPS). The birthday person can open the experience from any device, anywhere. Email auto-sending comes next.

Set up real automatic email sending

Luminary uses Resend — the cleanest email API available. Free tier: 3,000 emails/month, no credit card required.

Step 1 — Create a Resend account

1
Sign up at resend.com
Go to resend.com → Sign up free. Confirm your email.
2
Get your API key
In the Resend dashboard → API Keys → Create API Key → name it "Luminary" → click Create.

Copy the key. It looks like: re_aBcDeFgH_1234567890abcdef

Save it — you'll only see it once.

Step 2 — Add your domain (or use Resend's test domain)

Quick option (no domain needed): On Resend's free tier you can send from onboarding@resend.dev immediately — no domain setup required. Use this to test first.

For a proper sender address like birthday@yourdomain.com:

1
Add your domain in Resend
Resend dashboard → Domains → Add Domain → enter your domain name.
2
Add DNS records
Resend shows you 3 DNS records to add. Log into your domain registrar (Namecheap, GoDaddy, Cloudflare, etc.) and add them. Takes 5–30 minutes to verify.

Step 3 — Add the API key to Netlify

1
Open your site settings in Netlify
Netlify dashboard → your site → Site configuration → Environment variables → Add a variable
2
Add RESEND_API_KEY

Key

RESEND_API_KEY

Value

re_aBcDeFgH_your_actual_key_here
3
Add FROM_EMAIL

Key

FROM_EMAIL

Value (choose one)

# Option A — no domain setup needed (testing) Luminary <onboarding@resend.dev> # Option B — your own domain (professional) Luminary <birthday@yourdomain.com>
4
Redeploy your site
Netlify dashboard → your site → Deploys → Trigger deploy → Deploy site.

Environment variables only take effect after a new deploy.

Step 4 — Test it

1
Open your deployed send.html
Go to https://your-site.netlify.app/send.html, fill the form with your own email address as recipient, click Send.
2
Check your inbox
You should receive a beautiful branded email within a few seconds. The button inside opens the experience directly. If it goes to spam, add your domain's SPF/DKIM records in Resend.

Once this works: Clicking "Send the Experience" in Luminary will automatically send a branded email to every recipient. No manual clicking, no mail app, no copy-pasting. Just done.

How the email sending works

Here's the exact flow after you deploy:

1. You click "Send the Experience" in send.html ↓ 2. Browser calls POST /api/send-email (your Netlify function) ↓ 3. Netlify function receives: recipients[], birthdayName, experienceURL ↓ 4. Function calls Resend API with your RESEND_API_KEY ↓ 5. Resend sends a beautiful branded HTML email to each recipient ↓ 6. Recipient's inbox receives the email within seconds ↓ 7. They click the button → opens experience.html with their data ↓ 8. Success screen shows "✅ Sent to X recipients!"

If the API is not set up (local development), Luminary automatically falls back to showing Gmail / Mail app buttons so nothing ever breaks.

Optional: Custom domain

Replace luminous-birthday-abc123.netlify.app with luminary.yourdomain.com:

1
Add domain in Netlify
Netlify dashboard → your site → Domain management → Add a domain → enter luminary.yourdomain.com
2
Add CNAME record at your registrar
Type: CNAME Name: luminary Value: your-site-name.netlify.app
Wait 5–60 minutes for DNS to propagate. Netlify auto-provisions HTTPS.

How open-tracking works

EventHow it's trackedWhere stored
SentSender clicks "Mark sent" OR API returns successYour browser's localStorage
Openedexperience.html writes openedAt timestamp on loadSame browser's localStorage
ResponseBirthday person submits thank-you panelTheir browser's localStorage

Current limitation: "Opened" tracking only works if the recipient opens the link on the same browser/device you sent from (because localStorage is per-browser). For real cross-device tracking, a server endpoint is needed — see Future Improvements below.

Real open tracking (advanced)

To track opens across any device, add a second Netlify function that records opens server-side:

# netlify/functions/track-open.js # The experience.html calls: GET /api/track-open?id=WISH_ID # Store to: Supabase (free), Airtable (free), or Netlify Blobs (free)

This is a straightforward addition once the core deployment is working.

Troubleshooting

Email not sending — "API error"

# Check 1: Is RESEND_API_KEY set in Netlify? Netlify → Site config → Environment variables → look for RESEND_API_KEY # Check 2: Did you redeploy after adding the key? Netlify → Deploys → Trigger deploy → Deploy site # Check 3: Check function logs Netlify → Functions → send-email → View logs

Email goes to spam

Add your domain to Resend and verify the DNS records (SPF, DKIM, DMARC). Resend's dashboard shows exactly which records to add and confirms when they're verified.

Link still shows 127.0.0.1

You're still opening the file locally. Open the Netlify URL, not the local file. The URL must start with https://.

Camera doesn't work

Camera requires https://. Once deployed to Netlify, this works automatically. Never works on file://.

Function returns 500 error

# View real-time function logs: Netlify dashboard → your site → Functions → send-email → View log stream # Common causes: # - RESEND_API_KEY not set or incorrect # - FROM_EMAIL domain not verified in Resend # - Daily email limit exceeded (100/day free)

Photos not showing in the experience

Photos are encoded into the URL. If too many/large photos are uploaded, the URL exceeds browser limits. The size warning appears when photos exceed 800KB combined. Try 3–4 photos max, or smaller images.
